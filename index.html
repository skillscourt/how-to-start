Quick-start guide for developers 


          $ git clone https://github.com/lukezbikowski/skillscourt-vagrant
          $ cd skillscourt-vagrant
          $ vagrant up

Get yourself Yerba Mate (or just a coffee).
Your environment will be ready in few minutes.

What's happening now ?

  • Vagrant is getting you Ubuntu 14.04 LTS w/ git and rbenv
  • Installing ruby 2.1.2
  • checking out our github project
As ruby is build from source it might take a while...

What's next

Your code is in VAGRANT_DIRECTORY

Just bundle it!


          $ vagrant ssh
          $ cd /vagrant/skillscourt-api
          $ bundle install

Then startup the server


          $ bin/rails s
          $ cd /vagrant/skillscourt-api
          $ bundle install

The server is redirected to your host on 3030

Open localhost:3030 in your favourite browser!

Hosts configuration

To access our api you need to call subdomain api. To make it happen you need to add record to your hostname (Windows, MacOS, Linux) or use pow (MacOS) / prax (Linux)

Manual

Add the following line to the hosts file:

api.skillscourt.dev skillscourt.dev 127.0.0.1

Windows

To edit your host file you must open the file as Administrator
for example:

  • 'Hold SHIFT+right click on Notepad' -> run as Administrator
  • Open %SystemRoot%\system32\drivers\etc\hosts
  • Add api.skillscourt.dev skillscourt.dev 127.0.0.1 to the file.
  • To test open windows terminal and try: ping api.skillscourt.dev

Linux / MacOS / Unix

  • Open /etc/hosts as root.
    (for Ubuntu you can do sudo vim /etc/hosts)
  • Add api.skillscourt.dev skillscourt.dev 127.0.0.1 and save the file.
  • To test open terminal and try: ping api.skillscourt.dev
